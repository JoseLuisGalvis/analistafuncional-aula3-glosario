Aula N° 3 - Glosario Testing

Testing (Pruebas)
Proceso sistemático de ejecutar un programa o componente de software con el objetivo de encontrar errores y verificar que cumple con los requisitos especificados.
Prueba (Test Case)
Un escenario específico o conjunto de condiciones bajo las cuales se verifica un aspecto particular del software para determinar si funciona correctamente.
Defecto (Bug)
Una falla o error en el software que causa que funcione incorrectamente o de manera inesperada.
Prioridad (Priority)
Nivel asignado a un defecto o tarea de prueba para indicar su urgencia o importancia relativa en relación con otras tareas.
Criticidad (Criticality)
Gravedad de un defecto o incidente, que indica el impacto potencial en el sistema o en la operación del negocio.
Set de Pruebas (Test Suite)
Conjunto organizado de pruebas unitarias, de integración, de sistema, etc., diseñadas para probar un componente o sistema completo.
Plan de Pruebas (Test Plan)
Documento que describe los objetivos, alcance, estrategias, actividades, responsabilidades y cronograma de las pruebas de software.
Caso de Prueba (Test Case)
Conjunto de instrucciones paso a paso que describen cómo ejecutar una prueba específica para verificar un requisito o función del software.
